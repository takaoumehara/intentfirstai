Token ⑧ — Disclosure Dial

Disclosure Dial —
「何を知らせるか」の設計言語

プライバシー設定ではない。情報が流れる3つの方向を、人間関係ごとに制御する仕組み。

Flow 1 — Human → AI

自分がAIに何を教えるか

Sarahは食事制限をAIに共有する。でも家計の詳細は共有しない。ドメインごとに粒度が異なる。

Flow 2 — AI → Human (Connective)

AIが家族間の興味をつなげる

Kaiが嵐山の竹林に興味を持った。AIが祖父母に「お孫さん、竹細工に夢中でしたよ」と伝える——Kaiの許可範囲内で。

Flow 3 — AI → Others (Selective)

AIが外部に何を共有するか

アメリカのGrandparentsには英語でベスト写真だけ。京都の祖父母には全写真。同じ旅行でも、相手によって共有の形が変わる。

Design Constraint

Disclosure ≥ Autonomy — 知らせない × 任せるは不可能

AIにアレルギー情報を教えていなければ、レストランの自動予約は任せられない。Disclosureが低いドメインでは、Autonomy Dialも上がらない。これは設計上の制約であり、信頼構築の出発点。

Disclosure = Low
→
Autonomy = Suggest止まり
Disclosure = High
→
Autonomy = Notify / Auto可能

Phase 3 — Day 3-4 — Rain

AIは勝手に変えない。
選択肢を構造化する。

2日間連続の雨。屋外予定のリスケが必要。AIが屋内アクティビティを一覧化し、料金・予約・所要時間付きで提示。親がフィルタリングし、組み合わせプランを生成する。

Adaptation = 環境変化 → UIの再構成。天気が変わった。予定が使えなくなった。AIは勝手に変えない。代替案を構造化して、家族が選ぶ。

天気の変化が、プランのリコンポーズをトリガーする
予約していたメニューの魚が入荷しなかった。シェフが「今日はこの3つから選べます」と代替案を出す。メニューが変わっただけで、食事の体験は守られている。
ADAPTATION ⑥ Feasibility ④ Priority Weight ② Cognitive Load

Design Decisions

3つの方向 — P2 全シーンのマッピング

Context Grammarは3つの方向でAIの振る舞いを設計する。

DELEGATION

AIが構造化し、人間が決める

  • •家族投票（AIが構造化、家族が決定）

  • •体調不良（強制Confirm降格）
  • •AIが医療限界を認める
