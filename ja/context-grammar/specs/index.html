design-rules.yaml

デザインルール

33のif/thenルール。トークン状態をデザイン動作に変換する。3つの方向性: 委譲、適応、エスカレーション。

v2.0 · 33 rules · 3 directions
YAML 
rules:
  - id: R001
    name: cognitive_load_reduction
    when:
      cognitive_load: "> 0.7"
      form_factor: phone
    then:
      direction: delegation
      pattern: progressive_disclosure
      action: "reduce options to top 3, defer detail"

  - id: R002
    name: social_privacy_guard
    when:
      social_exposure: [public, authority]
    then:
      direction: adaptation
      pattern: social_aware_filter
      action: "suppress sensitive content, enlarge safe elements"

  - id: R003
    name: autonomy_escalation_check
    when:
      autonomy_dial: auto
      disclosure_dial: [hidden, existence]
    then:
      direction: escalation
      pattern: approval_gate
      action: "block auto-action, request confirmation"
      reason: "insufficient disclosure for autonomous action"

Integration

使い方

Context Grammarは機械に読まれるために設計されている。システムに応じた3つの統合パス。

01

エージェントフレームワーク統合

  • トークンスキーマをエージェントのコンテキストにロード
  • リアルタイムのトークン値を構造化入力として供給
  • ルールエンジンがコンテキストに応じた動作を生成
agent.load_schema("context-tokens.yaml")
agent.set_tokens({ cognitive_load: 0.8, form_factor: "phone" })
rules = agent.evaluate()

02

Claude Code / AIアシスタント

  • CLAUDE.md統合: プロジェクト指示にContext Grammarルールを追加
  • トークン対応プロンプティング: システムプロンプトにユーザー状態を含める
  • リアルタイムシグナルに基づくコンテキスト適応型レスポンス
User cognitive_load: 0.8
social_exposure: alone
→ respond concisely, skip formalities

03

デザインシステムブリッジ

  • デザインルールをコンポーネントバリアントにマッピング
  • トークン値をCSSカスタムプロパティに変換し、適応型UIを実現
  • 高い認知負荷でシンプルなレイアウトと大きなタッチターゲットを発動
--cg-cognitive-load: 0.8;
--cg-form-factor: phone;
→ .card { grid-cols: 1; font-size: 18px; }
