Context Grammar / Brain

記憶の3レイヤー。
ひとつの的確な応答。

行きつけのレストランは、来るたびにアレルギーを聞き直さない。Context Brainは、AIに同じ深さの記憶を与える — 更新速度の異なる3つのレイヤーとして構造化する。

Context Brain

覚えているレストラン

行きつけのレストランに入る。店長はもう名前を知っている。席の好みも。ナッツアレルギーも。これがLayer 1。

ウェイターは金曜はビール派だと覚えている。疲れた日は軽めを頼むことも。魚にはいつもレモンを追加することも。これがLayer 2。

そして今夜、上司と来ていることが見える。スーツを着ている。少し緊張している。これがLayer 3。

3つの異なる「知り方」。3つの異なる更新速度。ひとつのシームレスな体験。

今のAIにはこの構造がない。 Claude、Gemini、ChatGPTの記憶はフラットなメモ帳 — 階層も更新サイクルもアーキテクチャもない事実の羅列。Context Brainはそこに構造を与える。

Layer 1

Months to Years

Identity — あなたが誰か

最も遅いレイヤー。基盤。人生が変わるときに変わる — 気分が変わるときではない。

家族構成。食事制限。言語設定。アクセシビリティ。人生の「形」を決めるもの。日々の「質感」ではない。

What it holds 家族構成、食事制限、言語設定、アクセシビリティ、属性情報
Source ユーザーの明示的な入力 + オンボーディング
Update speed 月単位〜年単位

Restaurant Metaphor

「梅原様：ナッツアレルギー、窓際席、お子さん2人、記念日は3月」

予約台帳。一度書いて、めったに更新しない。毎回参照する。

Key Insight

このレイヤーの厚みはDisclosure Dialが制御する。フル開示ならシステムは完全なIdentityを得る。非開示モードなら名前だけ。AIにどこまで知らせるかはユーザーが決める。

Layer 2

Days to Weeks

Accumulated Learning — あなたが何をするか

信頼を獲得するレイヤー。あるいは失うレイヤー。

パターン記憶。生の事実から始まり、文脈的理解へと成長する。システムは観察し、相関を見つけ、行動モデルを構築する — インタラクションのたびに精度が上がる。

What it holds 食の好み、買い物パターン、スケジュール傾向、コミュニケーションスタイル、意思決定の癖
Source 行動からの暗黙的学習 + ユーザーの修正フィードバック
How it evolves 1回目：サーモンを注文（事実）
3回目：たぶんサーモンが好き（パターン）
10回目：金曜は疲れている、軽め＋ビール（文脈パターン）
20回目：妻がいるとワイン、一人ならビール（関係パターン）

Restaurant Metaphor

ウェイターの記憶。書き留めていない。反復から築かれる。修正で磨かれる。間違うこともある — でも良くなっていく。

Key Insight

L2は間違ったパターンを学習しうる。サーモンを3回頼んだのはゲスト用であって、自分が好きだからではない。修正フィードバックが軌道を修正する。誤学習が続くとAutonomy Dialが降格 — システムは確信度を下げ、より多く確認し、より少なく推定する。

Layer 3

Seconds to Minutes

Right Now — 今何が起きているか

最も速いレイヤー。呼吸するレイヤー。

リアルタイムフィード。現在のToken値、アクティブなIntent、デバイスの状態。コンテキストが切り替わるたび、センサーが読み取るたび、新しい情報がフレームに入るたびに変わる。

What it holds 現在のToken値（Physical State、Cognitive Load、Social Exposureなど）、アクティブなIntent、デバイス状態
Source センサーデータ + アクティブセッションのコンテキスト
Update speed 秒単位〜分単位

Restaurant Metaphor

「今日は急いでいる」「子供を連れている」「少し疲れている」

ウェイターの観察。履歴は不要。ただ目を開けているだけ。

Key Insight

L3は8 Context Tokensが住む場所。Physical State、Cognitive Load、Social Exposure、Priority Weight、Form Factor、Feasibility、Autonomy Dial、Disclosure Dial — すべてが現在の値をこのレイヤーに送り込む。リアルタイムの脈拍。

Special Pattern

Disposable Brain — 生まれて、学んで、返して、消える

すべてが永続する必要はない。京都への家族旅行に、AIの記憶の永久スペースは要らない。でもその旅行から得た学び — それは残すべきだ。

Disposable Brainは一時的な記憶インスタンス。ライフサイクルがある。

→

Step 1

生まれる

特定のコンテキストのために生成 — 旅行、プロジェクト、イベント。

→

Step 2

学ぶ

一時的なスコープ内でローカルパターンを蓄積する。

→

Step 3

返す

価値ある学びを永続的なHome Brainに送り返す。

Step 4

消える

一時インスタンスは削除される。学びは残る。

Trip Brain (Project 2)

京都 家族旅行

旅行のために生まれる。「カイがラーメンにハマった」「ミアは15時に電池切れ」を学ぶ。それらをHome Brainに返す。帰宅したら消える。次に京都へ行くとき、Home Brainはもうカイがラーメンを欲しがることを知っている。

Project Brain (Project 4)

ハリケーン保険対応

危機対応のために生まれる。トリアージパターンと判断ショートカットを学ぶ。判断パターンをElenaの永続Brainに返す。プロジェクト終了で消える。次の緊急事態では、彼女はもっと鋭い状態から始められる。

Design Principle

一時的なコンテキストには一時的な記憶を。ただし学びは必ず生き残る。Brainは消える。知恵は消えない。

Why It Matters

Brainは「記憶」ではない

Claudeにも記憶がある。Geminiにもある。ChatGPTにもある。どれもあなたについての事実を覚えている。

しかし事実はアーキテクチャではない。

フラットな記憶は「サーモンが好き」を覚える。

Context Brainは「リラックスした夜にサーモン、ストレス時は魚を避ける、デートで印象を良くしたいときにサーモンを頼む」を知っている。

違いはデータ量ではない。構造化されたデータ — 更新速度の異なるレイヤー、ドメイン別の軸、開示コントロール、時間的な認識。
