Context Grammar

エージェンティックAI時代のデザイン言語

スマホはメールを書き、フライトを予約し、会議を要約する。だが、あなたが疲れ切っていることは知らない。Context Grammar は、AIに人間の状態を読み取る言語を与える。情報ではなく、知性で応える仕組みである。

The Problem

誰も名前をつけなかった問題

AIアシスタントはリクエストの処理に長けている。だが、リクエストしている「人間」には盲目である。

01 — 観察

AIは驚異的なことができる

コードを書く。画像を生成する。旅行を計画し、カレンダーを管理する。毎月、できることリストは伸び続ける。技術は機能している。

02 — 謎

だが、毎回同じように応える

リラックスしていても、急いでいても。一人でも、上司が隣にいても。気軽に閲覧していても、重要な判断を下そうとしていても。同じ入力、同じ出力。なぜか。

03 — 発見

人間の文脈を伝える共通言語がない

デザイナーは「ユーザーはストレスを感じている」と言う。エンジニアは「ストレスのデータポイントは何だ？」と問う。誰も答えられない。概念レイヤーが欠落している。

Context Grammar は、その欠落したレイヤーである。人間の状態を、AIが読み取り、推論し、行動に変換できる構造化シグナルへ翻訳する8つのトークン。

Intent

ユーザーが何を求めているか

明示的な場合もあれば、暗黙的な場合もある。ユーザーが直接宣言することもあれば、トークンと記憶の交差点からシステムが推論することもある。Intentは起点となるシグナルである。

8 Context Tokens

6つの状況トークンと2つの関係ダイヤル

Physical State、Cognitive Load、Social Exposure、Priority Weight、Form Factor、Feasibility。さらに関係性を統御する2つのダイヤル：Autonomy DialとDisclosure Dial。これらが「いまのあなた」を記述する。

Context Brain

3つのレイヤーで構成される記憶

Identityは「あなたが誰か」。Accumulated Learningは「システムが時間をかけて観察してきたこと」。Right Nowは「この瞬間の真実」。記憶そのものはContext Grammarの専売特許ではない。その上に構築された構造化デザイン言語こそが独自性である。

Rule Engine

文脈がデザインルールに変わる

トークンと記憶がルールエンジンに入力され、具体的なデザイン判断が生まれる。何を見せるか、何を隠すか。どこまで自律に任せるか。どのモダリティを使うか。

Response

適応的な振る舞いの3つの方向

すべてのレスポンスは3つのカテゴリのいずれかに分類される。Delegation — システムが処理する。Escalation — システムが人間に判断を仰ぐ。Adaptation — システムが提示方法を調整する。抽象論ではない。実装可能なAXパターンに直結する。
