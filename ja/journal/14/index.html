エッセイ — 14

何を委ねるか、何を握るか。その境界線が動く理由

自律マップと自律ダイアルは別物だ。どちらも、信頼の蓄積とともにシフトする。

2025

あなたはすでに体験している

Netflix

あなたのNetflixのホーム画面は、友人のそれとまったく違う。

同じ映画。違う体験。あなたにはロマンスシーンのサムネイルが表示される。友人にはアクションシーン。Netflixは何十枚ものサムネイルを用意している。一枚ごとに、すでに判断を下している。「どの画像がこの人をクリックさせるか？」

あなたが視聴するコンテンツの80%はレコメンデーション経由だ。自分で選んでいるのは20%だけ。

あなたは自分で決めていると思っている。Netflixが決めている。

Amazon

いまAmazonを開いてみる。ホームページのすべての商品は、あなただけのために並んでいる。

「もう一度買う」を使ったことがあるだろう。先月、ドッグフードを注文した。ワンクリックで再注文。その前は：検索、スクロール、正しい商品を見つける、カートに入れる、確認。いまは：ワンクリック。

Amazonの売上の35%は、このレコメンデーションエンジンから生まれている。一日1,500億データポイントを処理する。あなたが次に何を買うかを予測するために。

Spotify

月曜の朝。Discover Weeklyを開く。30曲。聴いたことのない曲ばかり。なのに、どういうわけか、まさに聴きたかった曲ばかり。

最初の3曲を聴いて思う。「なぜ分かったのか？」

Spotifyはあなたのテイストプロファイルを一つだけ持っているのではない。複数持っている。ワークアウト中に聴く音楽と、日曜の朝コーヒーを飲みながら流す音楽は違う。だから複数のMixが届く。それぞれが、異なるバージョンのあなたに合わせてチューニングされている。

使えば使うほど、精度が上がる。

このパターンには名前がある

この三つのサービスには共通点がある。

AIに委譲されているもの：

```

Netflix → 何を観るかのレコメンド

Amazon → 過去に購入したものの再注文

Spotify → まだ知らない曲の選定

```

あなたが依然として決めているもの：

```

Netflix → 実際に観るかどうか

Amazon → 実際に買うかどうか

Spotify → 聴き続けるかスキップするか

```

委譲と所有の間の境界線。それが自律マップ（Autonomy Map）だ。

だが、委譲された側の中にも、レベルがある。

```

Netflix → ホーム画面を自動で変更する（高い自律性）

Amazon → 「もう一度買う」を表示するが、押すのはあなた（中程度）

Spotify → 知らない曲を追加するが、スキップは自由（中程度）

```

このスペクトラム──どこまで委譲しているか──が自律ダイアル（Autonomy Dial）だ。

マップとダイアルの正体

```

委譲 所有

情報収集 最終判断

・検索と比較 ・価値観に基づく判断

・パターン学習 ・金額の確認

・送信／公開

← 自律マップ →

```

自律マップ： 境界線をどこに引くか。何を委譲し、何を自分のものにするか。

自律ダイアル： 委譲されたものに対して、どこまで自律的にするか。四つのポジション：

```

提案 → 確認 → 通知 → 自動

「これで 「これで 「やりまし 完了。システム

いきますか？」 進めて た」 が完全に

いいですか？」 （取消可能） 処理

```

ツールを使う瞬間、委譲はすでに始まっている。Googleマップを開く？ ルート計算はすでに委譲している。だからこそ使う。マップなしのダイアルは無意味。ダイアルなしのマップは、ただ許可を求め続けるだけ。

名前をつけると見えてくるもの

Netflix、Spotify、Amazon──すでにこれを大規模に実行している。Spotifyはあなたのテイストの複数バージョンを理解している。Netflixはサムネイルのレベルまでパーソナライズしている。Amazonは売上の35%をレコメンデーションエンジンから得ている。Bettermentのようなロボアドバイザーは、ポートフォリオ全体を「自動」で運用している。

マップとダイアルは発明ではない。すでに存在するパターンのための「言語」だ。

では、なぜ名前をつけることが重要なのか。言語を持つと、まだ存在しないものが見え始めるからだ。

洞察1：ダイアルの天井は領域によって変わる

2021年、Netflixは「Play Something」（後に「Surprise Me」に改名）をリリースした。AIが番組を選ぶ。再生が始まる。入力不要。

利用率があまりにも急速に落ちたため、2年後に廃止された。

なぜか。エンタメの領域では、「何を観るか選ぶこと」自体が体験の一部だからだ。選ぶことを委譲すると、その価値を壊す。

一方、Amazonのスマートリオーダーは機能している。洗剤が少なくなると、自動で再注文される。誰も文句を言わない。洗剤を選ぶことは体験ではない。雑用だ。

```

領域ごとのダイアル天井：

日用品の補充： 天井 = 自動

→ 選ぶことに意味がない。Amazonスマートリオーダーが機能する理由。

エンタメ： 天井 = より良い提案

→ 選ぶことが体験そのもの。Surprise Meが失敗した理由。

→ AIの仕事は選ぶことではない。正しい選択肢を最初に浮上させることだ。

金融（ルーティン）： 天井 = 自動

→ 自動積立、自動予算管理、支出の自動カテゴリ分類。

金融（高リスク）： 天井 = 確認

→ 新規投資、大口送金、ローン。

```

注目すべき点。金融には天井が一つではない。毎月の自動積立は「自動」で動く。でも10,000ドルを新しいファンドに入れる？ あなたが決める。同じカテゴリ、異なるタスク、異なるダイアル設定。これは先行エッセイの四つの次元──可逆性、親密度、賭けの大きさ、コンテクスト──に接続する。ダイアルの天井はそれらの組み合わせだ。

洞察2：暗黙のダイアルと明示的なダイアルは同じではない

Netflix、Spotify、Amazonのレコメンデーションは自動的に改善される。あなたは何も設定していない。ただ使っているだけ。これが暗黙のダイアル（Implicit Dial）──観察を通じて上昇する。

Amazon Auto Buy？ あなたが設定する。「この商品の価格が15ドルを下回ったら自動購入」。意識的。意図的。これが明示的なダイアル（Explicit Dial）──自分でダイアルを回す。

```

暗黙のダイアル（行動を通じて上昇）：

→ Netflix：視聴するほどレコメンデーション精度が向上

→ Spotify：Discover Weeklyが毎週良くなる

→ ロボアドバイザー：学習によりポートフォリオのリバランスが最適化

明示的なダイアル（意図的に設定）：

→ Amazon Auto Buy：「15ドル以下で自動購入」

→ Tesla FSD：SlothモードかMad Maxモードを選択

→ Google Personal Intelligence：アプリごとにGmailアクセスを許可/ブロック

```

どちらが正しいか？ 両方必要だ。 ルールはシンプル：

```

低リスク、容易に可逆 → 暗黙のダイアルで十分

→ レコメンデーション、プレイリスト、検索品質

→ 間違い？ スキップするか、戻るだけ。

高リスク、取り消し困難 → 明示的なダイアルが必須

→ 購入、送信、予約

→ 間違い？ もう遅い。

```

洞察3：サービス横断の学習はまだ存在しない

Spotifyはあなたの音楽の好みを知っている。Amazonは買い物のパターンを知っている。Netflixはエンタメの好みを知っている。でもあなたを丸ごと知っているシステムは、一つもない。

月曜の朝、Spotifyでジャズを聴く。Amazonでプレミアムコーヒー豆を注文する。Netflixで料理番組を観る。一本の糸──リラックスした朝のリチュアル。でもその糸を見ているシステムはない。各サービスは自分のサイロの中だけで最適化する。

ここで「家政婦」の例が示唆的になる。

本物の家政婦はあなたの生活全体を見ている。冷蔵庫。パントリー。家族の好み。来週のスケジュール。一つのカテゴリを孤立して最適化しない。家全体にわたって学習する。それはSpotifyではない。Amazonでもない。

洞察4：開示ダイアル ── 委譲の前に、開示がある

ここまで見てきたすべてのパターンの下に、前提条件が埋まっている。

開示していないものは委譲できない。

Netflixのレコメンデーションは視聴履歴があるから存在する。Spotifyのプレイリストはリスニングデータから構築される。Amazonの「もう一度買う」は購入履歴を必要とする。サービスを使うこと自体が、暗黙の開示行為だ。

だが、これを家庭に持ち込むと、より意図的になる。家族の食事制限、好み、アレルギー、特定の嗜好──これらは利用ログからは見えない。明示的に共有する必要がある。

これを私は開示ダイアル（Disclosure Dial）と呼ぶ。領域ごと、人ごとの、AIにどこまで自分を知らせるかのコントロール。

開示ダイアルは、自律ダイアルの前提条件だ。開示が低い領域では、AIがどれほど高性能でも、自律性には自然な天井がある。十分に開示した領域では、より高い自律性が可能になるだけでなく、あなた自身がそれを望み始める。

続く家政婦の例で、この開示プロセスが時間とともにどう展開するかを見てほしい。

家政婦を雇ったと想像する

初日。

「砂糖はどこ？ どの塩のブランド？ 娘さんは牛乳を飲む？」

すべて、あなたが教えなければならない。買い物は委譲した。それが雇った理由だ。でも「何を買うか」──それはまだ完全にあなたの判断。

1ヶ月目：学習が始まる

```

マップのシフト：

委譲：玉ねぎのブランド、ヨーグルトの種類、補充タイミング

所有：今週の献立、特別な食材

ダイアル：

玉ねぎ → 提案（「また甘い玉ねぎでいいですか？」）

ヨーグルト → 提案（「いつもの3種類ですか？」）

```

3ヶ月目：家庭を学習し始める

```

彼女が気づいたこと：

あなたの妻：Trader Joe'sのハニーヨーグルト（これだけ）

あなた：無脂肪ギリシャヨーグルト（ブランドは柔軟）

子供：乳製品フリーのヨーグルト（どのブランド、どの味でもOK）

リブアイステーキ：このスーパーにはリブアイが2〜3パックあるが、

あなたに合う特定のパッケージは一つだけ。

他は合わない。

玉ねぎ：甘い玉ねぎがベース。売り切れなら、

黄色か白でいい。

紫の玉ねぎ？ 絶対にダメ。

ダイアルが上昇中：

ヨーグルト（3種類）→ 確認（「いつもの3つ、いいですか？」）

リブアイ → 確認（「見つけました。買いますか？」）

```

6ヶ月目：代替モードが出現する

ここから面白くなる。ダイアルが上昇するにつれて、新しいものが現れる。代替のルールだ。

レベルではない。モードだ。良い・悪いではない。異なる。アイテムごとに、AIは異なる指示を必要とする。

#### モード：フレキシブル（Flexible）

```

玉ねぎ → 甘い玉ねぎが標準。

売り切れなら、黄色か白でいい。

紫？ ダメ。

あなたのヨーグルト → 無脂肪ギリシャ。

ブランドはあまり気にしない。

そのタイプのギリシャヨーグルトなら何でもいい。

→ AIが代替して通知する。

→ 「甘い玉ねぎが売り切れだったので、黄色にしました」

```

#### モード：エグザクト（Exact）

```

妻のヨーグルト → Trader Joe'sのハニーヨーグルト。

この商品だけ。他の味ではない。

他のブランドでもない。

ただし：大サイズが売り切れなら、小6個はOK。

リブアイ → この特定のパッケージ。

他のリブアイはダメ。一切。

なければ買わない。

ビタミン剤 → このブランドだけ。

→ AIは代替しない。入手不可なら、買わない。

→ 事前承認された代替（サイズなど）だけが実行される。

```

同じモード、異なる柔軟性のレベル。妻のヨーグルト：「ブランドと味は絶対、サイズは柔軟」。リブアイ：「妥協ゼロ」。AIはこれらの微差を時間とともに学習する。

#### モード：エクスプローリング（Exploring）

```

プロテインパウダー → ホエイが標準。

でもいま積極的にテスト中。

先週：ダブルチョコレート。今週：バニラ。

来週：別のものを試す。

良い：バニラ（スムージーの味を邪魔しない）

悪い：ストロベリー、キャラメル（甘すぎる）

気になる：同タイプの別ブランド

→ AIがフィードバックを時系列で記録する。

→ 「前回バニラが好評でした。同じブランドの無味を試しますか？」

→ 一つに固定されない。選択肢をローテーションできる。

```

エクスプローリングモードは進行中のプロセスだ。積極的にテストし、フィードバックを与え、AIが学習する。最終的にフレキシブルになるかもしれない（「この3つをローテーション」）。エグザクトになるかもしれない（「これが勝者」）。モードは静的ではない。遷移する。

#### モード：サプライズ（Surprise）

```

日本のスナック → このカテゴリが好きだから、知らないものを持ってきて。

カリカリ。甘すぎないもの。

その範囲内なら何でもいい。

→ AIが基準内で未知の選択肢を選ぶ。

→ あなたの反応を記録する：「これは当たり」「これは外れ」。

```

でも、すべてにサプライズが適用されるわけではない。

```

日本のスナック → サプライズで ✓

ビタミン剤 → サプライズで ✗

プロテインパウダー → エクスプローリング（新しい味）、純粋なサプライズではない

```

サプライズはカテゴリごとのオプトインだ。サプライズが外れたとき、あなたはサプライズを減らしたくなるかもしれないし、増やしたくなるかもしれない。それはもう一つのダイアル──サプライズ自体の自律性だ。

モードは流動的だ：

```

エクスプローリング → 勝者を発見 → エグザクト（「ついにこれだ」）

エクスプローリング → 複数の勝者を発見 → フレキシブル（「この3つ」）

エグザクト → 飽きた → エクスプローリング（「新しいものを試そう」）

サプライズ → お気に入りを発見 → エグザクト（「これをもっと」）

フレキシブル → 好みが変わった → エクスプローリング（「いま何が食べたい？」）

```

好みは固定されない。季節とともに、気分とともに、エネルギーとともに、成長とともに変わる。AIは現在の好みだけを学ぶのではない。好みがどう進化するかを学ぶ。

1年後：完成したシステム

```

冷蔵庫は常にストックされている。

あなたは何もしていない。

何かが切れたとき → AIがモードに従う

新商品／価格変動 → 一時的に確認モード

サプライズ → 月に数回、新しいものが現れる

ある月、酸っぱいものが無性に食べたくなる。

AIがこのシフトを検知する。

あなたが何も言わなくても、サプライズカテゴリが

酸味の方向に傾く。

気づかないかもしれない。どのみち棚は変わり続けている。

これが「育てる」ということだ。

```

ダイアルは自動で上がるのか、自分で上げるのか？

暗黙のダイアル（Implicit Dial） ── 自動で上がる：Netflix、Spotify、Amazonのレコメンデーションは使うほど改善される。何も設定しない。あなたの行動がダイアルを回す。

明示的のダイアル（Explicit Dial） ── 自分で設定する：Amazon Auto Buy──「この商品を15ドル以下で自動購入」──意図的だ。Tesla FSDのモード、Google Personal Intelligenceの権限設定──自分で設定を選ぶ。

真実：両方必要だ。 リスクが決める：

```

低リスク、可逆 → 暗黙のダイアル

→ レコメンデーション、音楽、サムネイル

→ 間違い？ スキップして次へ。

高リスク、不可逆 → 明示的のダイアル

→ 購入、送信、確認

→ 間違い？ もう戻れない。

```

家政婦の場合：

```

暗黙：

→ 毎週あなたが甘い玉ねぎを選ぶのを見ている。

→ 翌週、聞かずに甘い玉ねぎを持ってくる。

明示的：

→ あなたが言う。「リブアイ。この特定のパッケージだけ。

他はダメ。いいね？」

→ それは明確な指示だ。

```

両方が同時に起きている。 観察が一つの信頼を築く。直接の指示が別の信頼を築く。合わさって、関係性が深まる。

全体像

```

① 自律マップ（境界線をどこに引くか）

「何を委譲し、何を自分のものにするか」

→ ツールを使う瞬間にベースラインが存在する

→ 時間とともに 所有 → 委譲 にシフト

② 自律ダイアル（委譲した中でどこまで）

範囲：提案 → 確認 → 通知 → 自動

→ 暗黙（使用を通じて上昇）

→ 明示的（自分で設定）

→ 例外時に一時的に下降

③ 代替モード（例外をどう処理するか）

委譲されたアイテムに対して、どんなパターンが支配するか？

→ フレキシブル／エグザクト／エクスプローリング／サプライズ

→ モードは時間とともにシフト

④ 開示ダイアル（AIにどこまで知らせるか）

「自分のどこまでをAIと共有するか？」

→ 領域ごと、人ごとに調整

→ 自律ダイアルの前提条件：開示していないものは委譲できない

→ 暗黙の開示（使用を通じてデータが蓄積）＋ 明示的な開示（あなたが伝える）

⑤ 信頼（四つすべてを時間とともに動かすもの）

→ 成功体験 → より広いマップ ＋ より高いダイアル

→ 失敗体験 → より狭いマップ ／ より低いダイアル

→ 想定外の事態 → 一時的なダイアル凍結

マップ = 何を

ダイアル = どこまで（自動か明示的か）

開示 = どこまで共有するか（ダイアルの前提条件）

モード = 各アイテムをどう扱うか

信頼 = いつ、どのくらいの速さで

```

巨大テック企業はいま、これをどう考えているか

Google ── マップとダイアルの両方を構築

Google Maps：

500回の通勤の後、あなたが見ている地図アプリは初日とは別物だ。開くと予測する。「職場まで20分」。聞く前に（暗黙のダイアル）。渋滞を迂回する。だが500回の通勤を経ても、まだ職場に向かって自動発進はしない。提案する。あなたが確認する。ダイアルにはまだ上昇の余地がある。

Gemini Personal Intelligence（2026年）：

GoogleはGeminiをGmail、フォト、YouTube、検索履歴に接続している。設計原則は明確だ。デフォルトですべてオフ。ユーザーがアプリごとにオプトイン（明示的ダイアル）。 Gmailは許可、フォトはブロック。あなたの選択。

GeminiのMemory機能は会話からあなたの好みを学習し、次回に適用する（暗黙のダイアル）。2026年2月、全ユーザーに無料公開された。

Apple ── プライバシーでマップを拡張する

Apple Intelligenceは約30億パラメータをデバイス上で直接実行する。処理のほとんどはオンデバイスに留まる。2026年1月、AppleはGeminiとの提携を発表。GoogleのモデルがAppleのPrivate Cloud Computeサーバー上で動く。「知能はGoogleから、データはAppleの境界の内側に」

プライバシー設計が、マップを拡張するエンジンだ。 データがデバイスから出ないなら、委譲への心理的抵抗が下がる。2026年後半、Agentic Siriがローンチ──メールからフライト情報を読み取り、Uberを自動予約する。そのとき、ダイアルが動く。

Samsung ── コンテクストを感知する家電

CES 2026：Bespoke AI Jet Bot Steam Ultra。ロボット掃除機がAI Object Recognitionを使う。人、猫、犬、ケーブルを認識する。AI Liquid Recognitionが液体のこぼれを検知する。そして判断する：掃除するか避けるか？ あなたのルールに基づいて（明示的ダイアル）。 自律的だが、ルールを書いたのはあなただ。

Kitchen AI Vision。Geminiが駆動し、冷蔵庫の中を見てレシピを提案する。「いま実際に作れるもの」のリアルな実装だ。

Microsoft ── マップを引くのはユーザーではなく組織

Copilot Control System。ここがひねりだ。自律マップを引くのは個人ユーザーではなく、組織のIT管理者だ。 Copilotがアクセスできるデータは何か？ 実行できるアクションは何か？ テナント単位、ユーザーグループ単位で設定される。個人はその境界の中でダイアルを調整する。

二つのマップが重なっている。組織のマップ、そしてパーソナルのマップ。

Amazon Alexa+ ── Auto Buyとコンテクストメモリ

Auto Buy： 明示的ダイアルの最も明確な実装。「Alexa、この商品の価格が100ドルを下回ったら買って」。意図的だ。アイテムごとに一つのリクエスト、最大200件アクティブ、24時間以内にキャンセル可能。

コンテクストメモリ：会話が積み重なる。月曜日に「今週はベジタリアンで」と言えば、木曜日のディナー提案が変わる。マップが拡張する（暗黙）。

Tesla FSD ── シナリオと個人によるダイアル

FSDはシナリオごとに調整する。低リスクの駐車場（ダイアル高め）、高リスクの横断歩道（ダイアル低め）。ドライビングスタイルを選べる（Slothモード vs. Mad Maxモード）──それは明示的ダイアル。

新しい点：Safety Score。急ブレーキの頻度、車間距離の維持、30日間のローリング平均で個人をスコアリング。まだFSDの自律性を直接変えるわけではない。だがインフラは整っている。実証された信頼に基づいてダイアルが上昇する基盤になりうる。

```

Google： ユーザーがマップを設定（明示的）

Memoryが会話から学習（暗黙）

Apple： プライバシーがマップを安全に拡張

Agentic Siriが2026年後半にダイアルを拡張

Samsung： ユーザーがルールを設定（明示的）

Geminiがキッチン AI Visionを強化

Microsoft：組織がマップを設定

ユーザーがその境界内でダイアルを調整

```

全社に共通するデザインの機会：

1. 段階的なマップ拡張 ── ツールを使い込むほど、「所有」が「委譲」にシフトする

2. 信頼に連動するダイアルの動き ── 成功した対話がダイアルを上げ、失敗が下げる

3. 代替モード ── 例外はすべて同じ扱いではない。アイテムごとに独自のルールがある

AIプロダクトチームが見逃す三つのこと

1. マップなしにダイアルを設定する

ほとんどのチームは「AIの自律性」を語るが、「何に対する自律性か？」とは問わない。ダイアルを見事に最適化しても、ユーザーが自分で握りたかった判断をAIに下させてしまう。マップがなければ、間違った瞬間に向けて設計することになる。

2. 暗黙のダイアルと明示的のダイアルを混同する

低リスクで可逆なもの（レコメンデーション）は暗黙で機能する。高リスクで不可逆なもの（購入、送信）は明示的が必要。区別がなければ、「聞かずにやった」か「どうでもいいことをいちいち聞いてくる」のどちらかになる。

3. 例外時にダイアルがリセットされない

在庫切れ、価格変動、想定外の事態。AIは一時的に確認モードに下がるべきだ。代替モードが設計されていなければ、例外が通常の瞬間と同じ自律性で処理される。信頼が壊れる。

AIプロダクトを設計しているなら

チームに三つの問いを投げかけてほしい。

問い1：「ユーザーが自分で握りたいものは何か？」

→ それが「所有」列だ。AIをそこに置くな。

問い2：「このダイアルは暗黙で上昇すべきか、ユーザーが明示的に設定すべきか？」

→ リスクと可逆性で判断する。

問い3：「信頼が蓄積されたとき、ダイアルはどう反応するか？」

→ 成功→ダイアル上昇のメカニズムを設計する。同時に、失敗→ダイアル下降のメカニズムも設計する。

一本の線

```

自律マップ = 境界線をどこに引くか

自律ダイアル = 委譲の中でどこまで

→ 暗黙（観察された行動）

→ 明示的（ユーザーが設定）

開示ダイアル = AIにどこまで知らせるか

→ 自律ダイアルの前提条件

→ 領域ごと、人ごとに異なる

代替モード = アイテムごとの例外処理

信頼 = 四つすべてを時間とともに動かすエンジン

マップなしにダイアルを回すな。

暗黙と明示的を混同するな。

例外をフル自律で実行させるな。

獲得された信頼なしに何も動かすな。

そして忘れるな：好みは変わる。

```

1年後、冷蔵庫は常にストックされている。あなたは何もしていない。

でも、棚に見慣れないブランドがあったら──それはAIだ。気に入らなければ、そう言えばいい。ルールが賢くなる。

それが人間とAIの関係だ。命令でも服従でもない。育てること。
