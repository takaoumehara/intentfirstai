# Hero Loop Illustration — Image Generation Prompt

## Concept
A seamless horizontal loop illustration showing the Context Grammar cycle:
**Intent → Context Reading → AI Interpretation → Experience → (loops back to Intent)**

The image should tile horizontally so it can be used as a CSS infinite scroll animation, creating a continuous loop that never ends.

---

## Art Style Reference (MUST MATCH EXACTLY)

Match the existing illustration style used throughout intentfirst.ai:

- **Perspective:** Isometric / 2.5D — consistent 30° axonometric projection
- **Human figures:** Solid black silhouettes with no facial features, simplified body proportions
- **Color palette:**
  - Primary: Cornflower blue (#4A7BF7 range) for structural elements, platforms, devices
  - Secondary: Warm amber/orange (#E89530 range) for accent elements, highlights, interaction points
  - Tertiary: Soft pink/rose (#E8A0A0 range) used sparingly for data/charts
  - White for facets and highlights on 3D objects
  - Light gray (#EDEDEB) flat background — NO gradients in the background
- **Object style:** Geometric, faceted 3D forms — like low-poly or paper-craft. Chunky, tactile feel. Not flat design, not photorealistic. Objects have visible facets and clean edges.
- **Line work:** Clean black outlines, medium weight, consistent stroke
- **Composition style:** Objects sit on small isometric platforms/pedestals. Connected by thin lines or dotted paths.
- **Devices:** Smartphones, TVs, watches drawn in the same isometric style with blue casings and visible screen content (simplified UI elements in orange/blue)

---

## The 5 Loop Stages (Left to Right, Seamless)

### Stage A — INTENT (Human Desire)
A black silhouette figure standing on a small isometric platform, with a **thought bubble** or **speech bubble** above their head. Inside the bubble: a simple icon representing a desire (a plate of food, a pair of shoes, a suitcase — pick one or show as abstract sparkle/star). The figure reaches forward or points toward the next stage.

**Visual metaphor:** A person with a clear want, about to speak it into existence.

**Connecting element → B:** A flowing path (dotted line or gentle arrow) leading from the person to the next stage. The path could have small blue geometric shapes floating along it (like data packets traveling).

### Stage B — CONTEXT READING (7 Tokens Sense the Situation)
A circular or hexagonal **sensor array** — imagine 7 small nodes arranged in a ring or arc around a central point. Each node is a small isometric cube or gem in alternating blue and orange. Above/below the nodes, tiny contextual icons float: a hand (Physical), a brain (Cognitive), an eye (Social), a scale (Priority), a dial (Synchro), a phone+TV (Form Factor), a clock+location (Feasibility).

**Visual metaphor:** The system is reading 7 dimensions of reality simultaneously.

**Connecting element → C:** The 7 nodes channel into a single converging stream (like 7 thin lines merging into one thicker arrow), feeding into the AI core.

### Stage C — AI INTERPRETATION (Context Grammar Engine)
A central **geometric polyhedron** (like the existing "AI" icosahedron in ImageE) — a faceted blue gem/crystal with "AI" or a sparkle inside, sitting on a larger platform. Orange arrows orbit around it. Small data visualizations (a mini bar chart, a routing line) float nearby. A black silhouette hand or arm reaches in, suggesting human oversight.

**Visual metaphor:** The grammar engine processes intent + context into a plan.

**Connecting element → D:** An outward burst — multiple paths fan out from the AI core toward different devices/surfaces.

### Stage D — EXPERIENCE (Adaptive Output)
Multiple **device surfaces** receiving the interpreted experience simultaneously:
- A smartphone held by a black silhouette hand (showing simplified swipe cards in orange)
- A TV screen on a stand (showing a different layout — bigger, more visual)
- A small watch or kitchen display (showing minimal info)

Each device shows **different content** — not mirroring. The phone shows 2 cards, the TV shows a gallery, the watch shows one line. This is role distribution.

**Visual metaphor:** One grammar, many surfaces, each adapted.

**Connecting element → E:** From the devices, a satisfaction indicator — a checkmark, a smile icon, or a completed action — floats upward and curves back toward Stage A.

### Stage E — LEARNING (Feedback Loop Back to Intent)
A subtle **upward spiral** or **curved return path** connecting back to Stage A. Along this return path: a small graph trending upward (the learning curve), tiny data points being absorbed back. A **circular arrow** or **loop indicator** makes it clear this cycles.

**Visual metaphor:** Every experience teaches the system. Day 1 → Year 1.

**Connecting element → A:** The path seamlessly connects back to Stage A's platform, completing the loop. The rightmost edge of the image should tile perfectly with the leftmost edge.

## Technical Requirements

- **Aspect ratio:** Ultra-wide, approximately 5:1 or 6:1 (e.g., 3000×600px or 3600×600px)
- **Seamless tiling:** The left edge and right edge must connect perfectly when placed side by side. Stage E's return path flows into Stage A's platform.
- **Background:** Flat light gray (#EDEDEB or similar) — the same neutral background as all existing illustrations
- **No text in the image** — all labels will be added via HTML/CSS overlay
- **High resolution** — at least 3000px wide for retina displays
- **Clean negative space** between stages — not overcrowded

---

## Mood & Tone

This is NOT a flowchart. It's an **experience illustration** — warm, human, optimistic. The loop should feel organic and alive, not mechanical. It should evoke:

- "Oh, this is a living system that gets better"
- "The person is at the center, not the technology"
- "It's continuous — not a one-time event"

Think of it as a **landscape panorama** that a viewer scrolls through continuously, like an infinite train window view of how Context Grammar works.
