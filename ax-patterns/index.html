Directions

Three Directions

Every pattern belongs to one of these three directions. Each direction is independent of the others — two would be insufficient, and a fourth is unnecessary.

D

Delegation

Gradually transferring authority to AI

Designing the process of progressively transferring decision-making and execution authority from humans to AI. Trust is not built overnight.

Human → AI
Consumer Approving ingredient substitutions every time → full delegation (Omakase Mode)
Enterprise Manager approves sales email drafts → routine patterns auto-sent
E

Escalation

Returning decisions to humans at AI limits

Designing mechanisms for AI to recognize its own limits and uncertainties, and return decisions to humans. It hands back control the moment a boundary is reached.

AI → Human
Consumer When a user reports feeling ill, explicitly state "I cannot make medical decisions" and suggest nearby clinics
Enterprise For an ambiguous translation at 71% confidence, display original text alongside candidate translations for human judgment
A

Adaptation

UI transforms in response to context changes

Transforming UI form, density, and content in response to contextual changes. Responds in real time to Token state changes.

Context ↔ UI
Consumer UI restructures for large screen when casting from phone to TV
Enterprise Dashboard auto-compresses to a 3-KPI summary view while on the move

Anti-Patterns

Anti-Patterns

Design prohibitions extracted from industry guidelines (Google PAIR, Microsoft HAX, Apple HIG) and brainstorming sessions.

AP1

Showing Confidence as a Percentage

"This suggestion has 73% confidence"

"73% confidence" creates "27% anxiety" for the user. Without the number, the anxiety wouldn't exist. Confidence should be conveyed indirectly through AI behavior.

Correct alternative: Autonomy Auto-Demotion changes behavior instead

AP2

Trying to Explain Everything

Always display the AI's full reasoning process

Showing the reasoning process causes users to re-evaluate every step, negating the point of delegating to AI. The explanation itself increases Cognitive Load.

Correct alternative: Progressive Disclosure. Default to conclusions only

AP3

Using Speed to Convey Confidence

Respond quickly when confident, slowly when uncertain

Zero industry evidence. Not adopted by Google PAIR, Microsoft HAX, IBM, or Apple. A slow response is simply interpreted as "slow."

Correct alternative: Autonomy Dial auto-demotion changes UI behavior

AP4

Same Transparency for All Users

Display the same level of reasoning trace detail for all users

Different expertise levels require different amounts of information. Even the same user has varying processing capacity depending on Cognitive Load.

Correct alternative: Calibrated Disclosure
