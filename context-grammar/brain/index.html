Context Grammar — Floor 3

Brain

AI memory is not for storing. It is for acting in this exact moment. Three layers, three speeds — that is the Context Brain.

The Problem

Every AI has memory.
None of them read "right now."

Gemini, Apple Intelligence, Claude — all of them remember your allergies, your routines, your preferences. None of them act on that memory in this exact moment. Memory is raw material. It becomes judgment only when crossed with right now.

The Restaurant

The reservation book says "nut allergy." It does not say whether the guest is in a rush, brought children, or is dining with their boss. Without that, the staff cannot deliver the right service.

Architecture

Three Layers

Memory updated at three different speeds, forming one living architecture

L3

Right Now

The waiter's observation

"They are rushing today." "They brought a child." Current values from the 8 Context Tokens flow into this layer in real time.

Update: seconds

L2

Accumulated Learning

The waiter's memory

"Last week the salmon was a hit." "Fridays, they are tired." The memory that sharpens with repetition, corrected over time.

Update: days / weeks

L1

Identity

The reservation book

Family structure, allergies, language settings. The bedrock that rarely changes. Written once, referenced every visit.

Update: months / years

Three Speeds

The reservation book (L1) changes once a year. The waiter's memory (L2) updates with every visit. The observation (L3) shifts by the second. A great restaurant reads all three simultaneously to deliver the right service. One layer alone is never enough.

Layer 1

Months to Years

Identity — Who You Are

Demographics, family structure, dietary restrictions, language, accessibility — the bedrock that defines the shape of your life, not the texture of your day. This layer changes when your life changes, not when your mood shifts.

What it holds Demographics, family structure, dietary restrictions, language preferences, accessibility needs
Source Explicit user input + onboarding
Update speed Months to years

The Restaurant Metaphor

"Mr. Umehara: nut allergy, window seat, 2 children, anniversary in March."

The reservation book. Written once. Updated rarely. Referenced every time.

Key Insight

The thickness of this layer is controlled by the Disclosure Dial. Full disclosure gives the system a complete identity. Hidden mode gives it a name and nothing else. The user decides how much the AI gets to know.

Layer 2

Days to Weeks

Accumulated Learning — What You Do

The layer that earns trust. Or loses it.

This is pattern memory. It starts with raw facts and graduates to contextual understanding. The system watches, correlates, and builds a behavioral model — one that gets sharper with each interaction.

What it holds Food preferences, shopping habits, schedule patterns, communication style, decision tendencies
Source Implicit learning from actions + user corrections
How it evolves Visit 1: ordered salmon (fact).
Visit 3: probably likes salmon (pattern).
Visit 10: tired on Fridays, wants something light + beer (contextual pattern).
Visit 20: orders wine when wife is present, beer when alone (relational pattern).

The Restaurant Metaphor

The waiter's memory. Not written down. Built from repetition. Refined through correction. Sometimes wrong — but getting better.

Key Insight

L2 can learn wrong patterns. You ordered salmon three times for guests, not because you love it. Correction feedback adjusts the trajectory. Persistent errors trigger Autonomy Dial demotion — the system reduces its confidence and asks more, assumes less.

Layer 3

Seconds to Minutes

Right Now — What's Happening

The fastest layer. The one that breathes.

This is the real-time feed. Current Token values, active intent, device state. It changes with every context switch, every sensor reading, every new piece of information that enters the frame.

What it holds Current Token values (Physical State, Cognitive Load, Social Exposure, etc.), active intent, device state
Source Sensor data + active session context
Update speed Seconds to minutes

The Restaurant Metaphor

"They are rushing today." "They brought a child." "They look stressed."

The waiter's observation. No history needed. Just open eyes.

Key Insight

L3 is where the 8 Context Tokens live. Physical State, Cognitive Load, Social Exposure, Priority Weight, Form Factor, Feasibility, Autonomy Dial, Disclosure Dial — all feeding their current values into this layer. This is the real-time pulse. See each token in detail →

Oxygen

Knowledge is frozen.
"Right Now" thaws it.

Until Right Now (L3) flows in, Identity and Accumulated Learning are static data. The moment "now" arrives, knowledge comes alive and action becomes possible.

Before — data only

L1 + L2

"Egg allergy" (L1) + "Tired on Fridays" (L2) = just data. The AI cannot act on this alone.

After — + Right Now

L1 + L2 + L3

+ "Tuesday 6:30pm, wife is in the kitchen, pasta and broccoli in the fridge, 25 minutes until dinner" (L3)

= "Egg-free pasta, 20 minutes, using what is in the fridge. Shall I proceed?"

The Restaurant

Staring at the reservation book does not produce a meal. The moment the staff sees how the guest walked in tonight — their pace, their expression, their company — service comes alive.

Lifecycle

Disposable Brain

Born for a purpose. Dissolved when it ends. The lessons survive.

Not everything should persist forever. A family trip to Kyoto does not need permanent shelf space in your AI's memory. But the lessons from that trip — those should survive.

Disposable Brain is a temporary memory instance with a clear lifecycle. See it applied in Project 02: The Family Trip →

01

Birth

The Trip Brain inherits what it needs from Home Brain: family allergies, preferences, past travel patterns. It arrives pre-loaded, not blank.

02

Learn

It builds its own knowledge during the trip. "Kai discovered ramen." "Mia crashes at 3pm." Insights that only exist inside this context.

03

Return

Valuable learnings flow back to Home Brain. "Kai likes ramen" becomes permanent knowledge. The trip's memory integrates into daily life.

04

Dissolve

The Trip Brain and its UI disappear. Plans are disposable. Memories are permanent. The context ends, but the wisdom stays.

Multi-Agent Orchestration

When AI Works in Teams

A single AI can handle your groceries. But planning a family trip to Kyoto? That requires a flight specialist, a hotel finder, an activity planner, and a budget tracker — all working together. Context Brain is what keeps them aligned.

Parallel Orchestration

Control Tower

One lead AI coordinates multiple specialists simultaneously. The Trip Brain sits at the center, sharing context — "the family has a nut allergy, the kids need breaks every 2 hours, budget is ¥300,000."

Each specialist works independently but reads from the same Brain. The lead AI resolves conflicts: "The hotel AI found a great ryokan, but the activity AI's temple schedule requires a different neighborhood."

When to use: Complex tasks with independent subtasks that share constraints.

Sequential Handoff

Field Relay

AIs work in sequence, each passing enriched context to the next. Research AI finds options → Booking AI reserves → Confirmation AI verifies with the family → Calendar AI schedules everything.

Each AI adds to the Brain before handing off. The Research AI writes "family prefers window seats" — the Booking AI reads it and acts accordingly.

When to use: Tasks with clear phases where each step depends on the previous one.

The key insight: Brain is the shared language between agents. Without it, each AI starts from zero. With it, the hotel AI already knows your daughter gets carsick — so it prioritizes walkable locations near the train station.

The Restaurant

A large party arrives. The sommelier, the chef, and the floor manager each have a role. But they all read from the same reservation book, the same waiter's notes. The sommelier knows about the allergy before the guest even sits down. No one asks twice.

Multi-Person Orchestration

One AI. One household. Four people — each with their own rules.

The same fridge, the same AI, the same shopping list. But each family member has a different Disclosure Dial setting, a different Autonomy level, a different Substitution Mode. The Brain tracks all of them simultaneously.

One AI, four rule sets

Mom: exact brand, auto-reorder. Dad: AI picks, confirmed. Daughter: discovery mode. Son: household default. One AI, four people, simultaneously.

One Brain, four parallel states

The Brain holds each person's token state independently. When rules collide — Mom wants the exact yogurt, the store is out — it knows whose preference wins.

Structure

Memory is not a flat list

Memory is organized along a domain axis — health, finance, hobbies, work — and a disclosure axis. Not every domain is stored at the same depth. Each accumulates at its own pace, and access scope differs per person.

The Restaurant

The waiter remembers everything about the guest's food preferences. But nothing about their work life. Each domain has its own depth of memory — and the guest controls which doors are open.
