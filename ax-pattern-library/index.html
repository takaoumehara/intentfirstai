What is AX Pattern Library

AIが読んだ文脈を、
どう返すかの設計言語。

Context Grammar の Input Grammar（8 Context Tokens + Brain）は「状況を理解する」語彙を与える。AX Pattern Library はその続き — 理解した状況に対して、AIがどう応答すべきかを体系化した21のインタラクションパターン集。Output Grammar の実装基盤。

→

Delegation

人間からAIへ、信頼を移す

成功体験の蓄積に応じてAIへの権限委譲を段階的に進める。委譲は遅いが、信頼崩壊は一瞬。Autonomy Dialの4段階（Suggest → Auto）で設計する。

←

Escalation

AIから人間へ、判断を返す

AIが自らの限界・不確実性を認識し、即座に判断を人間に返す。返しすぎると「使えないAI」、返さなさすぎると「暴走するAI」。閾値設計が核心。

⇄

Adaptation

文脈が変わり、UIが変形する

Token状態の変化にリアルタイムで応答し、UIの形態・密度・表示内容を変形させる。スマホからTVへ、1人から家族へ、昼から夜へ — 同じ情報でも器が変わる。

How to Use

8 Context Tokens がパターンを起動する

パターンは単独では発動しない。Context Tokenの状態変化がRule Engineを介してパターンのトリガー条件と照合され、UIの振る舞いが決まる。

STEP 01

Token状態を読む

Intent + 8 Context Tokens が現在の状況スナップショットを生成する。例：Cognitive Load = 高 × Autonomy Dial = Confirm。

→

STEP 02

Rule Engineが照合する

Token条件がパターンの発動条件と照合される。上の例では Cognitive Scaling が発動し、選択肢が3つに絞られる。

→

STEP 03

パターンがUIを変える

発動したパターンがUIの振る舞いを決定する。Approval Gateなら摩擦の重さが調整され、Form Factor Transformならレイアウトが再構成される。

→

STEP 04

Brainが学習する

結果はBrain（Identity / Accumulated Learning / Right Now）に記録され、次回の判断を更新する。成功体験が積み重なるほどAutonomy Dialが進む。
